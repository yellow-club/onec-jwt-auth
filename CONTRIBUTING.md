# Contributing

Спасибо, что хотите помочь ❤️

## Главное правило

**Clean Architecture first.**  
Если PR нарушает границы слоёв — его попросят переделать.

## Как делать PR

1. Создайте issue (bug/feature) — так проще договориться о дизайне
2. Форк → ветка → изменения
3. PR с описанием:
   - что меняется
   - почему так
   - как проверить

## Чек‑лист перед PR

- [ ] HTTPServices не содержат бизнес-логики
- [ ] Сценарии не ходят напрямую в регистры/справочники
- [ ] Добавлены/обновлены документы в `docs/`
- [ ] Нет логирования секретов (токены/пароли/secret PAT)

## Где искать “точки расширения”

- Добавление нового эндпойнта: см. `docs/ARCHITECTURE.md`
- Права/области: `ОперацииAPI`, `ПолитикаОпераций`, регистр `ПолитикиДоступаОпераций`
- Аутентификация: сценарии `СценарийАутентификация*`
