# Security заметки

- Не коммитьте реальные ключи/секреты (закрытый ключ RS256, PAT secret).
- В логах токены/секреты должны быть **REDACTED** — в проекте встроен санитайзер.
- Рекомендуется:
  - публиковать только по HTTPS
  - ограничивать публикацию AUTH (VPN/IP allowlist)
  - периодически чистить `JWTСессии` и отзывать компрометированные токены
- RS256: храните **закрытый** ключ только в ИБ/секрет-хранилище, а в `default.vrd` — **публичный**.

Если нашли уязвимость — пожалуйста, следуйте `SECURITY.md` (в корне репозитория).
